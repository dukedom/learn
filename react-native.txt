1.使用adb devices能查看到已连接到主机的移动设备，但RELOAD的时候依然红屏，使用 adb reverse tcp:8081 tcp:8081命令可以正常解决不能RELOAD的问题；
2.提示不能连接服务器另一种解决办法是，是摇晃手机 在出现的OPTION弹窗中选择DEV SETTING 配置目的主机的IP地址和端口号 例如:172.31.2.26:8081

3.关于RN中VIEW REF属性的意义：
ref 属性可以是一个回调函数，而不是一个名字。这个回调函数在组件安装后立即执行。被引用的组件作为一个参数传递，且回调函数可以立即使用这个组件，或保存供以后使用(或实现这两种行为)。

4.RN中的fetch方法是一个异步操作，在等待数据时可能会遇到常常获取OBJECT或undefiled的时候，这时候通过打印时间戳可发现fetch操作往往滞后，所以造成取不到数据的假象；

5.移动应用中常见的动态渲染局部或整张视图，这部分内容一定要参考组件的Props和State状态知识,使用this.setState方法即可局部渲染指定组件，但之前必须使用：
getInitialState: function() {
    return {
        newsDetail: null,
    };
  }
  对状态对象子项目值进行初始化；在fetch成功后设置setState状态值更新组件内容，调用setState方法后相当于重新调用组件的render方法进行渲染；
  新闻内应用 图片和文字数据接口是分开给的，而且现在这类应用的产品形式也都是图和内容分开展示的，这样开发端也好控制；
